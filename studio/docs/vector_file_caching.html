Vector File Caching

If file caching is enabled, the results of processing each data file are saved to temporary files within the nominated cache directory. This generally speeds up processing significantly when you make changes to the properties of some datasets or data files, since many of the data files within the project will not be affected by these changes. MapLink Pro will reload the processed data from the cache rather than reprocess the data if the data file properties have not changed since the last time the relevant layer was processed.

All MapLink Pro temporary files will be deleted from your temporary directory when you exit MapLink Pro Studio.

Because some of the MapLink temporary files may be very large (depending upon the size of the corresponding input data files), you should ensure that the disk on which the temporary directory is held has a reasonable amount of free space, and that the disk is not significantly fragmented.

In addition to a disk cache memory is also used. If the value specified is too small then the processing time of the vector data will increase considerably.

The recommendation is to allocate at least 1/4 of the memory in the machine to this cache.

For a 32bit version of MapLink Pro Studio you should make this 1/4 of the memory a 32bit process can address. The amount available to a 32bit process will depend on the Operating System you are running MapLink Pro Studio on.
