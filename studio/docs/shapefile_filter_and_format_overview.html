Shapefile Filter and Format Overview
SHP Filter and Format Overview

The SHP Filter allows MapLink Pro Studio to read data supplied in Shapefile format. Shapefile format data consists of three files, having the extensions .shp, .shx and .dbf.

In general, Shapefile data is supplied in themed files. Each set of files contains features relating to a theme. An example of a theme could be "Roads". The themed files would then contain features of class "Road".

Some Shapefile data is supplied with each file set containing a mixture of features of different classes, where the feature class of each feature is defined in a field in the .dbf database. In this case it becomes necessary to extract the feature class of each primitive from the relevant database field. Facilities are provided in the Shapefile Configuration Panel to handle this case.

.shp File Description

The .shp file contains the data's geometrical information. MapLink Pro Studio supports the following geometrical primitives used in the Shapefile format:

Point

Multipoint

Arc (actually a polyline)

Polygon

There is no text primitive in Shapefiles.

.dbf File Description

The .dbf file associated with the .shp file contains a database of arbitrary information in dBase IV format. Each record in the .dbf file is associated with a single geometrical entity in the .shp file. MapLink Pro Studio allows the information in the database to be used to define levels of subclassing, create text primitives with position or attach field values to geometrical primitives as names.

The Shapefile Filter can be configured on a file-by-file, or dataset-by-dataset basis. In both cases the Shapefile Configuration Panel is used.
