VPF Filter and Format Overview

The VPF filter enables MapLink Pro Studio to read data in VPF format, such as VMAP0 data.

VPF data is always opened from a file called "lat" (with no extension). This file references the other files containing the data.

Each VPF data file references one or more "libraries". Typically these might contain a "reference library" (such as outline data for the whole world) which can be used to place the detailed data in the other libraries in context, plus one or more data libraries, which might for example contain data for a particular geographical area at various different resolutions.

The VPF format definition is used to define a number of geographic data products:

Additional Military Layers, AML (AML is also available in S-57 format)


Digital Nautical Chart, DNC

Digital Topographic Data, DTOP

Foundation Feature Data, FFD

Tactical Ocean Data, TOD : Level 0,1, 2 and 4

Urban Vector Map, UVMap

Vector Interim Terrain Data, VITD

Vector Map, VMap : Level 0, 1 and 2

World Vector Shoreline Plus, WVSPlus0


A common VPF data format is VMAP0, which is explained in more detail in the VMAP0 section.

VPF datasets are normally configured on a file-by-file basis, although they can be configured on a dataset-by-dataset basis (except for the choice of library). This is done using the VPF Configuration Panel.

Note that the VPF filter automatically discards attribute data whose value is "No Entry Present".

In addition to supporting the VPF datasets, MapLink Pro Studio also includes the standard Geosym Symbol Library.
