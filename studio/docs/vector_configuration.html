Vector Configuration

The Vector Configuration tab of the Data File Property Sheet contains those configuration options that are specific to vector data files (a separate Raster Configuration tab will appear for raster data files).

Currently the only common processing option available for all vector data files is whether to apply polygon keyholing to the input data.

For data source filters that provide embedded attributes in the MapLink Pro primitives the panel also allows the user to create configurable labels and names for entities based on the attributes stored with the MapLink Pro primitives. ( Note: as of MapLink 3.4.1.2 only the GDF+ filter provides embedded attributes for features ).

Label generation is only available for line features which have embedded attributes. Label generation is only available when the panel is being applied to datasets.

Entity Name generation is available for all features that have embedded attributes.

Entity Name Configuration

The Entity Name configuration panel allows the user to select a feature and to give each entity of that feature a name which is configured according to the embedded attributes of that feature.

In the example above ( taken from GDF data ) all Road Elements of type FC_0 are being given a name based upon the route number ( RN ) and offical name ( ON ) attributes which are included with the feature.

Entity names are generated using the same Label Definition Syntax as the Label Generation.

Entity names are useful when interrogating a map. This behaviour can be seen by moving the mouse cursor over map in MapLink Pro Studio and observing the entity information ( which includes the name ) on the right hand side of the status bar.

Note: the embedded attributes of a particular feature type depend on the source dataset. To make use of this feature an understanding of the source data attributes is important.
