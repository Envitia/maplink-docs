GDF Filter and Format Overview

The GDF+ Filter allows MapLink Pro Studio to read data supplied in GDF format. GDF format data consists of a single ACSII file with the extension GDF which follows the European standard format for describing road networks and transferring road related data. It is widely used for car navigation systems and in many other transport and traffic applications such as traffic management and control, fleet management, dispatch management, traffic analysis, and automatic vehicle location.

In general, GDF data is supplied covering a specific geographic area. Each file may contain a number of different feature types within the geographic area.

.gdf File Description

The GDF file contains the data's geometrical information. MapLink Pro Studio supports the following geometrical primitives used in the GDF format:

  • Point
  • Line
  • Polygon

Each feature has associated with it a number of attributes which the GDF+ Filter will analyse.

There are no text primitive in GDF files though the GDF+ Filter will provide text primitives for point features with an "ON" attribute and a selected set of area features which have the "ON" attribute

Address Area

  • Park/Garden
  • Industrial Area
  • Built-up Area
  • Island
  • Water Element

The GDF Filter can be configured on a file-by-file, or dataset-by-dataset basis. In both cases the GDF Configuration Panel l is used.
