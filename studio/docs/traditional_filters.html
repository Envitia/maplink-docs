Traditional Filters

There are four traditional filtering techniques are available in MapLink Pro Studio. All four methods act on Polygons and Polylines, and require the geometric model to be decomposed into a set of vertices. This can be a slow process for large amounts of data. Points may only be removed if they satisfy the following conditions:

  • The point is not the start point or end point of a Polygon or Polyline
  • The point is shared between 2 Polygons at most. If a point which is shared by 3 or more Polygons was to be removed, then there would a danger of white space appearing between the Polygons.

Some data is provided with slight errors between points that should be considered co-incident. If this is the case, a Map Unit tolerance can be provided. This allows two points whose separation is less than the tolerance to be treated as co-incident.

MapLink Pro Studio also provides an evaluation helper to help decide which values are sensible for layer based upon the current data loaded.

If you wish to apply filtering, you may be able to improve the overall map performance further by optimising the data for compression.
