MIF Filter and Format Overview

The MIF Filter allows MapLink Pro Studio to read data supplied in MapInfo Interchange Format (MIF). MIF format data consists of two files, having the extensions .mif and .mid.

In general, MIF data is supplied in themed files. Each .mif/.mid file pair contains features relating to a theme. An example of a theme could be "Roads". The themed files would then contain features of class "Road".

Some MIF data is supplied with each .mif/.mid file pair containing a mixture of features of different classes, where the feature class of each feature is defined in a field in the .mid database. In this case it becomes necessary to extract the feature class of each primitive from the relevant database field. Facilities are provided in the MIF Configuration Panel to handle this case.

.mif File Description

The .mif file contains the data's geometrical information. MapLink Pro Studio supports the following geometrical primitives used in the MIF format:

Line

Point

Polyline

Rectangle

Region (actually a polygon)

Text

.mid File Description

The .mid file associated with the .mif file contains a database of arbitrary information. Each record in the .mid file is associated with a single geometrical entity in the .mif file. MapLink Pro Studio allows the information in the database to be used to define levels of subclassing, create text primitives with position or attach field values to geometrical primitives as names.

The MIF Filter can be configured on a file-by-file, or dataset-by-dataset basis. In both cases the MIF Configuration Panel is used.
