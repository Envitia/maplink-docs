ASCII Filter and Format Overview

The ASCII Filter allows the user to add data to a project using a very simple ASCII format file. It could be used to add annotations or graticule lines that are not provided in the source data.

The file format is very simple. Currently the ASCII filter supports four primitive types; simple polygons, lines, points and text. The file should be saved with the extension .ascii.

The format consists of a file header and version number followed by an unlimited number of primitive definitions. Coordinates can be specified in any arbitrary coordinate system. Sizes should be specified in units of the chosen input coordinate system.

Each primitive definition starts with the primitive type, and the feature code string associated with the primitive. The primitive type determines the format of successive lines within the primitive definition.

Parameters enclosed in square brackets ‘[]’ are optional.

File Header

TSLASCIIMAP 104

Simple Polygon Primitive Definition

POLY "<FeatureCodeString>" ["<ObjectNameString>"]

<X1> <Y1>

<X2> <Y2>

:

<Xn> <Yn>

Polyline Primitive Definition

LINE "<FeatureCodeString>" ["<ObjectNameString>"]

<X1> <Y1>

<X2> <Y2>

:

<Xn> <Yn>

Point Primitive Definition

POINT "<FeatureCodeString>" ["<ObjectNameString>"]

<X> <Y> <RotationDegrees> [<ProjectionAngularAdjustment>]

The 'ProjectionAngularAdjustment' value can be either the string true or false. The default value assumed is false.

  1. 'true' means that the rotation provided is adjusted for the output projection of the map.
  2. 'false' means that the rotation is applied as specified with no correction applied.

Text Primitive Definition

TEXT "<FeatureCodeString>" ["<ObjectNameString>"]

<X> <Y> <RotationDegrees> <HorizontalAlignment> <VerticalAlignment> <Size> "<Text>"

Horizontal Alignment values are:

  1. Left
  2. Centre
  3. Right
  4. Undefined

Vertical Alignment values are:

  1. Full Bottom
  2. Baseline
  3. Middle
  4. Capital Top
  5. Full Top
  6. Undefined

If the ‘Undefined’ value is used in the ASCII file then the alignment defined in the Feature Book is used instead. If the ASCII file says 'undefined' and the Feature Book says 'defined by filter' then the default is Left, Full Top.

The “<Size>” parameter is used to calculate the extent of a text primitive. This extent is used in conjunction with the active clipping routines to determine whether or not the primitive should be removed from the current project. The “<Size>” should be specified in units of the chosen input coordinate system.

“<Text>” may contain the C newline escape sequence ‘\n’. The backslash can also be escaped as: ‘\\’. Text in version 103 of the file format is expected to be UTF-8 encoded.

To create an ASCII format file

  • Open a text editor
  • Add the header TSLASCIIMAP 104 as the first line in the file
  • Add primitive definitions according to the specification above. See Example
  • Save the file with the extension .ascii

To use an ASCII format file

  • Add an new ASCII dataset to the layer
  • Add the .ascii file to the dataset
  • Reload the layer, if automatic data loading is not enabled.
