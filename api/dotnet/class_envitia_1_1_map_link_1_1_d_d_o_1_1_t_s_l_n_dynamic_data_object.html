Detailed Description

This abstract base class represents a generic object from the 'real' world. These objects may also have different visual representations of themselves (through TSLNDisplayObject class objects). Derived classes will represent concrete real world objects (such as e.g. Accidents, Roadworks, ...).

TSLNDynamicDataObject (DDO) objects should be added to a TSLNObjectData Layer. Once added to it, the object data layer then assumes ownership of the DDO object & the client need not worry about deleting those DDO pointers.

When a TSLNObjectDataLayer is added to a TSNLDrawingSurface, a TSLNDisplay Object is created for each DDO by calling instantiateDO on the DDO class. Derived-DDO classes should return an appropriate TSLNDisplayObject in their implementation of this method.

pure virtual

Called to create the TSLNDisplayObject (DO) to be used for this instance of the TSLNDynamicDataObject (DDO).

Parameters
keyOne of TSLNDisplayType values. Determines the type of visualisation manager to use. Currently always tslNTMS.
dsIDThe ID of the TSLNDrawingSurface to which the DDO is being added.

The dsID allows clients to supply different classes of Display Object for different TSLNDrawingSurfaces, or provide Drawing Surface specific positions for the Display Objects. This assumes that the client has tagged all drawing surfaces in his app with different (known) ids.
