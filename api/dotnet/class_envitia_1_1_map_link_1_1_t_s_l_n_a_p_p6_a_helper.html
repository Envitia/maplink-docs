Detailed Description

The TSLNAPP6AHelper class should be used to manipulate the APP6ASymbols. It provides methods for creating symbols as a Bitmap, Entity Sets or TSLNAPP6ASymbol objects.

Either the app6aConfig.csv or 2525bConfig.csv configuration files are required for this class to operate correctly. These files define the symbols for the APP6A or 2525B specifications respectively. These files can normally be found in your MapLink home config folder. During construction of a TSLNAPP6AHelper instance an attempt will be made to find and load the app6aConfig.csv file at the default location. If you wish to load the alternate configuration file, specify the full path to the file, including the filename, to the constructor.

Note: There is no internal reference counting provided within the internal implementation; each instance of this call will attempt to load the config file once.

You can check that the configuration file has been successfully loaded by calling the 'valid()' method after construction.

Check the TSLNErrorStack::lastError() method for a description of the problem.

◆ defaultConfigPath

String^ Envitia::MapLink::TSLNAPP6AHelper::defaultConfigPath
staticgetset

Use this method to get/set the default location of the configuration file.

When setting this value the user can either provide the path to a directory, in which case the class will attempt to load the app6aConfig.csv file from that directory, or the full path to the configuration file to use including filename.

When a TSLNAPP6AHelper class is constructed it will first check if a path has been passed in the constructor and attempt to load that. However, if no path has been passed, the default config path will be used to locate the symbols.
