Detailed Description

Windows NT specific implementation of the abstract TSLN2DDrawingSurface class.

This class is the .NET equivalent of TSLNTSurface (GDI) and TSLDXAcceleratedSurface (DirectX Accelerator SDK) classes in the C++ API.

This method allows the TSLNDrawingSurface to draw to a HDC to which it is not attached.

Typically this method would be used to request the TSLNDrawingSurface to draw to a printer device context.

Note: the device co-ordinate system for the target HDC may not be the same as the one currently attached to the TSLNDrawingSurface. Because of this, it may be necessary to do a temporary wndResize() and resize() on the TSLNDrawingSurface.

See Also: TSLNDrawingSurface::setDeviceCapabilities

Parameters
hDCHandle of the device to draw to.
(x1,y1)Bottom left co-ordinate (in user units) of the area to draw.
(x2,y2)Top right co-ordinate (in user units) of the area to draw.
clearIf true this will clear the specified area before drawing any of the data.

Color colour,
Bitmap ^ bitmap,
OUT String ^% fontName,
String ^ outputString,
OUT String ^% sectionName )

Query example text rendering used by TSLNDrawingSurface.

This method populates the specified bitmap with a sample of the specified text style. The bitmap is not cleared before drawing the text. The height of the bitmap is used define the height of the text. The text displayed is either the specified outputString, or the font name if no string is passed.

The name of the font will be returned through the fontName parameter.

Parameters
indexIndex of a font style
colourColour to display the text
bitmapBitmap to populate with text
fontNamePopulated by the base name of the font
outputStringString to be displayed, otherwise font name is used
sectionNameThis will be populated with the section heading that this font style appears under
Returns
a true on success, false otherwise.

◆ setupDynamicColours()

bool Envitia::MapLink::TSLNDrawingSurface::setupDynamicColours ( String^ filename)

Load colour definition file into the surface dynamic colour list.

This method allows the user to set up a list of colours which are used only by this instance of the Drawing Surface.

The current dynamic colours are not cleared. The colours loaded will overwrite any colour index that already exist.

If addPathList or loadStandardConfig has been called then the pathlist is used to search for the file in suitable directories (using keying), if not then the system Path and the environment variable MAPLINK_PATH are used to search for the file, or alternatively the filename may be fully qualified.

Parameters
filenameName of the colour file.
Returns
true on success, false otherwise.

◆ symbolStyleValue() [1/4]

bool Envitia::MapLink::TSLNDrawingSurface::symbolStyleValue ( int index,
Color colour,
Bitmap^ bitmap )

Query example symbol rendering used by TSLNDrawingSurface.

This method allows the user to populate a bitmap with a symbol, according to an index into the symbol list used by the drawing surface. The aim of this method is to give the user the ability of viewing the symbols used by the drawing surface.

The user creates the bitmap to be populated. The size of the bitmap is typically 32*32 pixels if the symbol is an icon.

If the symbol style index refers to a raster symbol, then the scalability of the symbol will be dictated by the symbol configuration file (TSLNRasterSymbolScalableAsSymbolFile).

If the symbol style index refers to a font symbol, then this method won't draw anything to the bitmap. Use one of the other symbolStyleValue methods instead, providing the character to draw.

Parameters
indexIndex of a symbol style.
colourColour of the symbol.
bitmapBitmap to populate with symbol rendering
Returns
true on success, false otherwise.
