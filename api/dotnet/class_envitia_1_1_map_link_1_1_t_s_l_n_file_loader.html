Detailed Description

File loader interface class.

TSLNFileLoader serves as a base class for more specialised file loaders. It provides the basic methods for loading data from local or remote files.

See description of the load() method for more details.

◆ load()

TSLNLoaderStatus Envitia::MapLink::TSLNFileLoader::load ( TSLNPathList ^ pathList,
String ^ filename,
TSLNLoaderCallback ^ callback,
Object ^ arg,
bool synchronous )

Load data from a file, synchronously or asynchronously, with progress callbacks.

If successful, the data is returned in a stream as part of the callback function.

If the file does not exist, a failure callback is sent and the routine returns a failure.

There will always be a callback signifying the end of the load, which may be LoadingOK with done=100%, or one of the other status codes. One of these termination callbacks will always be sent for each call made to load().

Asynchronous loads are queued and happen one after another.

If the callback is made with status = LoadingOK and 100% done, the stream parameter will be valid. Note that the return value from the callback is ignored for this last callback made.

Note that when the call is asynchronous, the callback will be made from another thread, not the thread used to call load(). The user is responsible for safely synchronising the callback into the main application using thread-safe mechanisms, message passing etc., see the sample code for an example.

Parameters
pathListlist of directories to search for file. May be null. Files are searched for in the directories listed and in the local directory, or the filename may be an absolute path.
filenamefile to find.
callbackfunction to call with progress and completion information, and to return the data when reading is successfully completed.
argargument passed to callback function, user-defined.
synchronousIf true then this call will only return once the load is complete, has failed or been cancelled. If false then the call returns if the file exists and continues the load in a background thread.

Returns a status denoting the current process of the load. The load may not have completed yet depending upon the value passed in the synchronous parameter.

◆ setLoadedCallback()

void Envitia::MapLink::TSLNFileLoader::setLoadedCallback ( TSLNLoaderFileLoadedCallback^ callback,
Object^ userData,
bool callBeforeDecompression )

This method allows the user to set a callback function which will be called for every file loaded by this TSLNFileLoader instance.

The callback will occur before the data is processed by MapLink Pro allowing the user to abort the use of the stream and to erase small parts of the stream.

The envisaged use is where the user has added a CRC to the files, where on each file load, the CRC is checked and potentially removed from the stream.

To disable the callback pass a null as the callback argument.

Parameters
callbackA user supplied function that implements the TSLNLoaderFileLoadedCallback delegate. This will be called with userData when the file has been loaded but before it has been processed.
userDataA pointer to user supplied data. This will be forwarded to the user supplied callback function.
callBeforeDecompressionIf true then the callback will be called before the data has been compressed (if it was compressed). If set to false the callback will be called after the data has been compressed.
