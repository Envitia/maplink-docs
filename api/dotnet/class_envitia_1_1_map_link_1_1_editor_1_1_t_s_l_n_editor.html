Envitia::MapLink::Editor::TSLNEditor Class Reference
MapLink.Editor.NET

Detailed Description

Generic editing component for MapLink Geometry objects. This component should be attached to one drawing surface at a time, using the attach method. Events such as button presses should be passed to this from the application.

An instance of this class must be paired with an object derived from TSLNEditorRequest. The methods of the request object are invoked by the TSLNEditor component when information needs to be requested from or sent to the user.

NOTE : The editor may force redraw requests to the screen if an operation requests it.

◆ addOperationAlias()

bool Envitia::MapLink::Editor::TSLNEditor::addOperationAlias ( String^ originalName,
String^ aliasName )

Create an operation by cloning an existing operation.

An operation alias will allow an existing operation to be invoked using a different name.

This is useful when combined with User Operations to provide multiple implementations of essentially the same operation. For example, an application may require several kinds of polyline to be drawn, each with a different feature code. The operation alias mechanism may be used to create different named operations which all perform the same original polyline functionality, with the advantage that overriding user-defined operations can modify the feature code after the polyline has been created.

Note that the current implementation at the time of the method call is used, regardless of subsequent operation additions. Thus, if the basic TSLNPolylineOperation is added, the addOperationAlias called and a user-defined polyline operation added, then the alias would use the basic TSLNPolylineOperation rather than the user-defined polyline operation. If the addOperationAlias call occurred after the user-defined polyline operation had been added, then that would be used as the basis of the alias.

Parameters
originalNameName of original operation to clone.
aliasNameNew name of the operation, used when activating.
Returns
true if the operation is successful aliased, false otherwise.

◆ addUserOperation()

bool Envitia::MapLink::Editor::TSLNEditor::addUserOperation ( TSLNUserOperation^ op,
String^ name,
TSLNOperationType type )

Add a new user-defined operation to the editor.

The user-defined operation is defined by an instance of a class derived from the TSLNUserOperation class. When the editor receives an activation request for the named operation, the TSLNUserOperation is activated in turn.

A user-defined operation can override an existing operation by specifying the same name. For example, if the client application wished to override the "polygon" operation to perform some action after the polygon had been created, then it should add the normal polygon operation (TSLNPolygonOperation::add( editor )) then add a user-defined operation of the same name - "polygon". The methods of the user-defined operation would then be called in place of the original operation methods. To invoke the original operation methods from within the user-defined operation methods, call the 'invokeOriginal' method of the TSLNUserOperationRequest handler.

Note that this can be combined with an addOperationAlias call to provide multiple implementations of essentially the same operation. For example, an application may require several kinds of polyline to be drawn, each with a different feature code. The operation alias mechanism may be used to create different named operations which all perform the same original polyline functionality, with the advantage that overriding user-defined operations can modify the feature code after the polyline has been created.

Parameters
opPointer to user defined operation to add to the editor.
nameName of the operation, used when activating.
typeType of operation - primitive, manipulation or attribute.
Returns
true if the operation is successful added, false otherwise.

Informs the editor that the current data layer has changed.

This informs the editor that the current editable data layer has changed in the current drawing surface. This will cause the editor to reset back to the select operation, clear the select list and to request the current entity set from the data layer.

◆ FollowMode

bool Envitia::MapLink::Editor::TSLNEditor::FollowMode
getset

Whether follow mode is enabled.

If enabled, then certain landlink API operations will follow the edges defined by existing primitives. The specific operations affected are:

  • "polyline" (TSLNFollowPolylineOperation)
  • "polygon" (TSLNFollowPolygonOperation)
  • "replacesection" (TSLNReplaceSectionOperation)
  • "topologicalreplacesection" (TSLNTopologicalReplaceSectionOperation)

This can be toggled on and off midway through drawing a line/ring.
