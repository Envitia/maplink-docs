Detailed Description

Specialist data layer for the handling of non-map data.

The Standard Layer component is provided to allow the user to load, create, manipulate and save non-map data.

The methods provided by the TSLNStandardDataLayer are divided into a number of categories:

  • Loading and saving data
  • Manipulation of data
  • Rendering
  • Miscellaneous

The methods 'loadData', 'loadDataFromBuffer', 'append' and 'appendDataFromBuffer' allow data to be loaded from files or data streams. 'removeData' removes all data from the data layer. The data stream is expected as a pointer to a memory buffer of unsigned char *. The methods 'saveData' and 'saveDataToBuffer' allow the user to save all the data in the layer to a file or a data stream. A buffer is often used when the data is subsequently saved to a database.

Entities may be cloned from other data layers using the methods 'cloneEntity', 'cloneExtent' and 'cloneFeature'. Entities may be removed from the TSLNStandardDataLayer using the methods 'removeEntity' and 'removeFeature'. The ordering of the entities may be modified using the methods 'bringToFront' and 'sendToBack'.

The rendering of features within the TSLNStandardDataLayer can be modified using the methods 'loadRendering', 'saveRendering', 'setRendering', 'getRendering' and 'clearRendering'. Rendering information held by the TSLNStandardDataLayer is used for any entities that do not have any individual rendering attributes.

A number of miscellaneous methods and properties are also available.

Add a feature to the feature class list.

This method adds a feature to the feature class list. This list may subsequently be queried but has no effect on the display of data.

If the specified feature_id already exists in the feature list, then the name of that feature is changed to the feature_name passed. No limitation is placed on features sharing the same name, but this is not recommended.

String ^ detail_layer_name,

int feature_id )

Clone entities with specific feature_id.

This method allows the user to clone all entities with the given feature from the source TSLNMapDataLayer to this TSLNStandardDataLayer.

A detail layer name within the TSLNMapDataLayer must be specified, along with an extent to request.

For TSLNStandardDataLayers, both the extent and detail layer name parameters are ignored. All entities with the feature id specified will be cloned.

Parameters

(xmin,ymin)Bottom left corner of extent to clone.
(xmax,ymax)Top right corner of extent to clone.
feature_idID of the features to be cloned.
Returns
true if any entities were cloned, false otherwise.
