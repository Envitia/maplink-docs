Detailed Description

This is the NITF Filter DataLayer.

This layer permits the import of both the vector and raster data in the order specified in the dataset.

This layer is keyed.

◆ setConfigFile()

bool Envitia::MapLink::TSLNNITFFilterDataLayer::setConfigFile ( String ^ file)

NITF requires the use of its configuration file in order to load data onto an instance of the filter data layer. If the user does not provide the location of this configuration file then it will automatically be loaded from the default location from the MAPL_HOME variable. On machines with an installation of MapLink, this will be the NITFconfiguration.ini file in the config directory of the installation. If this file cannot be accessed, then loading will be unsuccessful.

This filename must be set before the loadData method is called on the data layer. Calling this method after calling loadData will cause the new configuration location only to be used on subsequent loadData calls.

Parameters
fileThe full or relative path and filename of the configuration file. Passing null will cause the previously set configuration file to be cleared and will return to using the default.
