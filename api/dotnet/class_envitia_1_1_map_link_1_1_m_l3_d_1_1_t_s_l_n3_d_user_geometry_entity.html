Detailed Description

Specialist entity for the handling of user-defined 3D geometry objects.

The 3D User Geometry Entity allows the developer to create an object that can be stored inside a TMF file and drawn as part of the drawing of a 3D Standard Data-Layer.

The developer should at minimum derive from TSLN3DClientUserGeometryEntity in the application and implement the pure virtual draw method. Additional methods on TSLN3DClientUserGeometryEntity may be overridden as necessary.

Only the methods defined on TSLN3DClientUserGeometryEntity are supported for a user defined Entity at present. This means that setting of attributes etc. is not supported.

This extension is purely for rendering of objects defined by the user.

It is the developers responsibility to ensure thread safety.

When the position or size of the user geometry changes it is necessary to call the updateBoundingBox() method on the TSLN3DEntitySet that contains the associated TSLN3DUserGeometyEntity. This ensures that the viewport culling performed on the TSLN3DEntitySet uses the current bounding sphere of the user geometry.

Note: If you are not using the version of Visual Studio that MapLink was built with then you need to manage the deletion of the TSLN3DClientUser GeometryEntity derived object yourself.

This function stores a pointer to the callback function to the User defined Load function that reads, creates and returns the 3D User Geometry object.

You can register more than one callback. Once registered you can not remove a callback.
