MapLink Pro Developer Toolkit » MapLink.Net

Detailed Description

This class represents the definition of an attribute of a feature. This includes the type, multiplicity, precision, name and other relevant properties of the attribute.

MapLink does not enforce the properties defined in the TSLNFieldDefinition on the relevant feature attributes.

◆ encoding

getset

Set or retrieve the encoding of the TSLNFieldDefinition. Passing a NULL value is invalid and will result in no changes occurring.

This value is defaulted to an empty string.

The encoding property is intended as a metadata field, used to denote the source attribute type.

◆ enumeration

TSLNVariant^ Envitia::MapLink::TSLNFieldDefinition::enumeration
getset

Set or retrieve the enumeration values in the field definition.

It is intended that the variant type of the provided enumeration value should match the type specified in the TSLNFieldDefinition. The value contained by the enumeration variant should be an array of the permitted possible values that attributes of this field may contain. For instance if the field definition is defined to be of type TSLNVariantTypeDouble then the enumeration property should be set to be an array of doubles.

Alternatively, a TSLNVariant of the type TSLNVariantTypeEmpty or TSLNVariantTypeNull implies that the TSLNFieldDefinition does not contain an enumeration value.

By default, the enumeration contains a TSLNVariant of type TSLNVariantTypeEmpty.

◆ key

String^ Envitia::MapLink::TSLNFieldDefinition::key
getset

Set or retrieve the key in the TSLNFieldDefinition.

This value should be specified as a 2 character code which uniquely identifies this field within the scope of the specified featureID.

This value should be set before the TSLNFieldDefinition object is passed to the TSLNDataHandler.

◆ maxExclusive

TSLNVariant^ Envitia::MapLink::TSLNFieldDefinition::maxExclusive
getset

Set or retrieve the "maxExclusive" value stored in the field definition.

It is intended that the variant type of the provided maxExclusive value should match the type specified in the TSLNFieldDefinition. The value contained by the maxExclusive variant is used to denote the smallest value, larger than the allowed range, which is not permitted.

Alternatively, a TSLVariant of the type TSLNVariantTypeEmpty or TSLNVariantTypeNull implies that the TSLNFieldDefinition does not contain a maxExclusive value.

By default, the maxExclusive contains a TSLNVariant of type TSLNVariantTypeEmpty.
