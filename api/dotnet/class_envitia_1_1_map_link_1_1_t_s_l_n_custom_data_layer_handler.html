Detailed Description

Specialist class providing additional functions for use when a TSLNClientCustomDataLayer implementation is being drawn. This class is not user constructable.

This class provides a means for TSLNClientCustomDataLayer derived classes to draw additional TSLNDataLayers not present in a TSLNDrawingSurface as part of their normal drawing.

Allows a MapLink data layer to be drawn as part of the drawing of a TSLNClientCustomDataLayer. A TSLNDataLayer derived class is constructed and populated as normal by a TSLNClientCustomDataLayer implementation and passed to this method to be drawn to the drawing surface as part of the rendering of the TSLNClientCustomDataLayer.

Data layers drawn in this fashion should not have been added to the drawing surface as this will result in the layer being drawn twice.

By default a data layer drawn in this fashion inherits the data layer properties of the TSLNCustomDataLayer that draws it. This can be overridden by passing a TSLNCustomDataLayerProperties instance containing the desired data layer properties in the layerProperties argument.

Limitations

  • Multiple data layers may be drawn in a single TSLNClientCustomDataLayer::drawLayer() call, however all layers drawn this way will use the active TSLNCoordinateSystem for the drawing surface for all coordinate transformations. Thus if two TSLNMapDataLayers are being drawn they must use the same TSLNCoordinateSystem for both layers to draw correctly.
  • Transparency for data layers drawn via this method will only function if the parent TSLNCustomDataLayer has been set up for transparency to work. For more information see the documentation for the TSLPropertyEnum enumeration.
  • Data layers drawn via this method will use the TSLPropertyBuffered setting of the parent TSLNCustomDataLayer regardless of the value set on the TSLNCustomDataLayerProperties instance passed in.
  • The TSLNPropertyDetect, TSLNPropertySelect and TSLNPropertyTransparent data layer properties have no effect on data layers drawn via this method.
Parameters
dataLayerThe TSLNDataLayer to draw to the drawing surface containing the TSLNCustomDataLayer. The user retains ownership of this data layer.
extentThe extent in TMCs of the data layer to draw. In most cases this will be the same as the TSLNEnvelope passed to the TSLNClientCustomDataLayer::drawLayer() method.
layerNameThe name of the layer being drawn.
layerPropertiesAn instance of the TSLNCustomDataLayerProperties class that defines the data layer properties to use when drawing the layer. For information on the available properties see the documentation for the TSLNPropertyEnum enumeration. The user retains ownership of this object.
Returns
true if the layer was drawn successfully, false otherwise.
