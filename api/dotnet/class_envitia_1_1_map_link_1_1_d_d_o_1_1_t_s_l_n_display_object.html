Detailed Description

This abstract base class represents the visualisation of a TSLNDynamicData Object (DDO). Derived classes will represent concrete display objects (such as e.g. AccidentDO, RoadworksDO, ...).

Derived-TSLNDisplayObject (DO) classes must provide an implementation of the draw() method, which is actually responsible for painting the object on screen.

When a TSLNObjectDataLayer is added to a TSLNDrawingSurface, a DO is created for each DDO by calling the instantiateDO method associated with that DDO class.

A derived-DDO class can have several classes of DOs associated with it. This allows clients to use different visualisation in different drawing surfaces.
