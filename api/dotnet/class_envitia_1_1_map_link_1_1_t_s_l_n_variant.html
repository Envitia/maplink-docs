Detailed Description

This class encapsulates a discriminated union for use with the TSLN... classes. It can hold instances or arrays of the following set of base types:

  • TSLNVariantTypeBool (bool),
  • TSLNVariantTypeChar (8bit char),
  • TSLNVariantTypeShort (16bit short),
  • TSLNVariantTypeLong (32bit integer),
  • TSLNVariantTypeFloat (32bit float),
  • TSLNVariantTypeDouble (64bit double),
  • TSLNVariantTypeStr (string),
  • TSLNVariantTypeNull (null (empty) string - different from a 'nil' variant which can apply to any type).
  • TSLNVariantTypeEntityID (64bit integer - TSLEntityID)
  • TSLNVariantTypeULong (32bit unsigned integer)
  • TSLNVariantTypeBinary (binary blob)
  • TSLNVariantTypeDateTime (System::DateTime)
  • TSLNVariantTypeCode (An enumeration value and code space)
  • TSLNVariantTypeMeasurement (A numeric measurement and unit of meaure)
  • TSLNVariantTypeReference (string)
  • TSLNVariantTypeURI (string)
  • TSLNVariantTypeVariant (A collection of other TSLNVariants)

These types are defined by the enumeration TSLNVariantType.

NOTE:

  1. TSLNVariantTypeChar stores an array of characters, TSLNVariantTypeStr stores an array of strings. Since all arrays have a maximum size of 255, TSLNVariantTypeChar is limited to 255-character strings. Use an array of strings to store arbitrary-length strings (the array may have one entry, to store a single string).
  2. For single strings, 'String^ getStringVal' is the easiest for querying. The easiest way to deal setting string data, or arrays of string data, is to use the setStringVal / getStringVal / getStringLen. methods rather than the overloaded getVal / setVal methods.
  3. A null string ("") is treated as a non-array (isArray() returns false) of type TSLNVariantTypeNull - getStringVal handles this correctly.
  4. All arrays are limited to 255 entries maximum.
  5. For compatibility with DEC-Alpha which has 8-byte longs, longs are treated as 32-bit integers by this class.
  6. Variants may be 'nil' - which can be tested or set using the 'isNil' method. If this is true then the variant value should not be queried.
  7. Variants may be external references - such as a database key or hyperlink. If so, then the 'isExternalReference' flag is set

◆ getStringVal()

String ^ Envitia::MapLink::TSLNVariant::getStringVal ( )

Get the value if a string value is held. To obtain the value as a string even if the variant holds a non-string type, use getValueAsString.

If the variant is of type CHAR, or NULL, or a single STR, returns the string, otherwise fails. The string is truncated if necessary according to the maximum length specified (this is the length of the buffer passed, including space for the null terminator). A suitable value can be obtained by calling getStringLen if the buffer is to be dynamically allocated to avoid truncation.

Returns a valid string if the variant is type CHAR or a STR with size 1, null otherwise.

Converts the contents of the variant to a string using simple formatting.
