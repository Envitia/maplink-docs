TSLTerrainVSAlgorithm::Parameters Class Reference

Detailed Description

This object is used to store the parameters for a viewshed operation.

Parameters are required for all viewshed operations: -The input data object -The compositor object

Optional Parameters: -One or more location filters -User-supplied data(void*)

Modifying the member variables of this class will set the default setting for a viewshed operation. Any parameters which are not passed to TSLTerrain VSAlgorithm::calculateViewshed may be set directly on this class. Whether a parameter is used or not is defined by the viewshed algorithm implementation, as not all parameters will apply to all algorithms.

This parameter object is made available to the other objects during the viewshed operation. An application may pass additional data through the m_ userData attribute.

Member Data Documentation

◆ m_endHeight

double TSLTerrainVSAlgorithm::Parameters::m_endHeight

This offset will be applied to all 'end' points used by the viewshed calculation. This may be used to calculate the visibility of objects, which are not located on the ground.

Depending on the value of m_endHeightRelativeToGround, this value may be absolute, or relative to the terrain height at the end point.

◆ m_endHeightRelativeToGround

bool TSLTerrainVSAlgorithm::Parameters::m_endHeightRelativeToGround

If this flag is true, the end height will be considered relative to the terrain height of the centre point. If this flag is false, the end height will be considered relative to zero.

◆ m_maxRadius

double TSLTerrainVSAlgorithm::Parameters::m_maxRadius

The maximum radius of the viewshed calculation.

Sample points, which are further than this distance from the centre point, will not be processed. The corresponding points in the output data object will not be modified.

◆ m_minRadius

double TSLTerrainVSAlgorithm::Parameters::m_minRadius

The minimum radius of the viewshed calculation.

Sample points, which are less than this distance from the centre point, will not be processed. The corresponding points in the output data object will not be modified.

◆ m_startHeight

double TSLTerrainVSAlgorithm::Parameters::m_startHeight

The height of the current viewshed operations centre, in units defined by the input data.

Depending on the value of m_startHeightRelativeToGround, this value may be absolute, or relative to the terrain height at the centre point.
