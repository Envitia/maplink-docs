Detailed Description

This is the base class for the specific Filter DataLayers.

Filter DataLayers provide access to the MapLink Filters so that applications can process data directly.

The process of importing and processing data is very similar that used by MapLink Studio.

Additionally the Filter DataLayer design allows importing of both vector and raster data from a dataset in the order specified in the dataset if the filter supports this functionality (this is not possible in MapLink Studio).

Some of the Filter DataLayers are keyed. Please contact your sales representative for additional information.

By default all data is processed.

◆ getGeodeticClipping()

bool TSLFilterDataLayer::getGeodeticClipping ( bool & applyGeodeticClipping,
double * minLatitude,
double * maxLatitude,
double * referenceMeridian,
double * degreesWest,
double * degreesEast ) const

Geodetic clipping allows the user to clip data using a shape defined by lines of latitude and longitude upon the earth's surface. By contrast, output clipping clips against a rectangle in the defined output coordinate system, and in general will not clip against lines of latitude and longitude. In geodetic clipping, the extent in latitude is defined normally using minimum and maximum values, but the extent in longitude is defined in terms of a reference meridian and extents West and East of the reference meridian. This removes any potential ambiguity in whole world datasets. The reference meridian is always included in the clipped extent.

This function retrieves the values currently set on the data layer. If the applyGeodeticClipping value is set to false then the other arguments will be ignored and not set.

Parameters
applyGeodeticClippingSpecifies whether a geodetic clipping should be applied. This can be used to turn off a previously set clipping.
minLatitudeThe minimum latitude to clip from.
maxLatitudeThe maximum latitude to clip to.
referenceMeridianThe reference meridian from which the degreesWest and degreesEast arguments are taken from.
degreesEastThe minimum value from the reference meridian to clip from.
degreesWestThe maximum value from the reference meridian to clip to.
Returns
true on success, false otherwise

◆ process()

bool TSLFilterDataLayer::process ( )

Processes all the data immediately rather then waiting until the data layer is drawn.

Processing must take place before the loaded data may be drawn. If the processing has not taken place before the data is drawn, then processing will automatically take place at display time. This method allows the processing to take place before display time.

Display items that have been set to not be visible will not be processed. Display items that have not changed since they were last processed will not be reprocessed.

Returns true on success, else false.

◆ processDisplayItem()

Forces a display item to be processed or reprocessed. Whereas the process() method will skip display items that have not changed since they were last processed or items that have had their visibility set to false, this method will force the display item to be processed. Display Items that have have changed since processing will return false when their validity is queried.

◆ rasterOptions() [1/2]

bool TSLFilterDataLayer::rasterOptions ( const int bitDepth = 0,
const int numberOfColours = 256 )

This method sets the bit depth of the output rasters to that specified.

The bit depth can either be set to 8 (colour index) or 24 (RGB).

If the bit depth is 0 then the default output will be that of the input data.

In addition if the bit depth is set to 8 then the number of output colours can also be set.

The valid range of output colours is 0..256.

If the colours are limited to a specific number the colours output in each layer may be different.

Parameters
bitDepthbit depth for output rasters
numberOfColoursnumber of colours to limit the output rasters too.
Returns
true on success, false otherwise.

◆ rasterSplitThreshold() [1/2]

unsigned int TSLFilterDataLayer::rasterSplitThreshold ( ) const

This method returns the threshold (in Megabytes) that if a raster exceeds the raster will be split into strips or rectangles.This is to reduce the overall memory usage whilst processing, but can make processing take longer.

The threshold is defined in Megabytes and a value of 0 implies that a raster is not to be split.

bool TSLFilterDataLayer::setDirectories ( const char * attachmentDirectory,

Set the directories to where data is stored when data has been loaded on a TSLFilterDataLayer

This call should be made before loading data. If the data layer currently has data loaded then this call will return false.

Parameters
attachmentDirectoryDirectory where data attachments are written to. Then the data is unload or the data layer is destroyed then these attachments files will be deleted. All temporary files will be deleted when the data layer is destroyed or the data removed. By default, if this value has not been set then the operating system specific temporary directory is used.
Returns
true on success.
