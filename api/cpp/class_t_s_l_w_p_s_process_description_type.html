Detailed Description

Represents the WPS process description, it encodes the ProcessDescription Type element from the WPS Schema. A user implemented WPS plugin should return a populated instance of this class when the describeProcess method is called.

◆ statusSupported() [1/3]

const bool * TSLWPSProcessDescriptionType::statusSupported ( ) const

The status supported property is an optional property. If not present, the implied value (default) is "false". If present, it indicates whether or not ongoing status information can be provided for this process. If set to "true", the status element of the execute response doucument shall be kept up to date.

This is acheived by informing the WPS framework of the status at ir/regular moments determined by the user implemented plugin. The user implemented plugin should use the TSLWPSProgressSink passed into the executeProcess for this purpose.

Returns the statusSupported property of the process description. If none has been set this method returns 0.

◆ storeSupported() [1/3]

const bool * TSLWPSProcessDescriptionType::storeSupported ( ) const

The store supported property is an optional property. If not present, the implied value (default) is "false". If present, it indicates whether or not complex data outputs from this process can be stored by the WPS server as web-accessible resources.

If storeSupported is "true", then the execute method may include "as Reference" equals "true", indicating that the WPS client wishes to utilise the server's results storage mechanism. See TSLWPSStoreHelper for further details.

Returns the storeSupported property of the process description. If none has been set this method returns 0.
