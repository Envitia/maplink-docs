Detailed Description

DirectX9 Accelerated drawing surface for Microsoft Windows.

Creates the TSLAcceleratedRenderer when the user adds a class derived from TSLAccleratedRenderControl.

MapLink Notes:

The draw extent will be ignored and the whole display will be redrawn on a draw.

The translucency (TSLPropertyTransparency) of the first buffered layer defines the translucency of the tiled rendering. Rendering of the buffered tiles also occurs at this point.

Layers which are single buffered will be drawn in the order they were added or subsequently reordered. Unless the layer is an Accelerated Custom layer the drawing will be performed using GDI. In order not to impact too much on performance do not interleave DirectX and GDI layers too much.

Single buffered layers interleaved with double buffered layers will be drawn after the buffered tiles have been drawn.

When using an Accelerated Custom Datalayer then view, world and projection matrices need to be preserved.

The getRenderingInterface() call returns a TSLRenderingInterface for the HWND.

DirectX Notes:

You can only use HWND's when attaching or creating a DirectX Surface.

MapLink sets the D3DCREATE_FPU_PRESERVE flag when creating a device. This is required because MapLink needs to use doubles for accurate calculation of data.

Portions of Direct3D assume floating-point unit exceptions are masked. Unmasking such exceptions may lead to undefined behavior. The C/C++ runtime libraries mask all floating-point exceptions by default.

The C/C++ math defaults are:

  • Rounding: NEAR
  • Precision control: 53 bits
  • Interrupt exception mask: INVALID, ZERODIVIDE, OVERFLOW, UNDERFLOW, INEXACT, DENORMAL

Limitations

The method wndResize can not be used to draw to a smaller area of the window.

TSLDXAcceleratedSurface::TSLDXAcceleratedSurface ( HWND wnd,
TSLDXSwapBuffer swapEffect )

Construct a Windows DirectX specific TSLAcceleratedDrawingSurface. MapLink will construct a sensible default DirectX device. If you require additional control then use the other Constructor.

This particular setup will do all the setup for DirectX.

Parameters
hwndHWND to which the TSLAcceleratedDrawingSurface is to be attached.
swapEffectThis is the DirectX swap effect.

The swapEffect controls how the back buffer is displayed onto the screen. Normally you would want to use TSLD3DSwapBufferDiscard or TSLD3DSwapBuffer Flip as these would redraw all the buffer. The size of the back buffer is the size of the window. MapLink can be told to only draw to part of this buffer by a call to TSLDrawingSurface::wndResize(...), however on the swapping of the buffers the whole buffer is displayed on the screen.

If you only want to redraw part of the Screen you will want to use TSLD3DSwapBufferCopy. This really only makes sense if you are using the EditorSDK or you are doing something similar to the EditorSDK and are only doing partial updates to the screen.

Notes

TSLAcceleratorConfiguration.swapBuffers(false) and any drawing should have the clear flag set to true.

If you wish to draw before the MapLink surface you need to do the following

// Set the surface configuration up.
TSLAcceleratorConfiguration config;
surface->getConfiguration(config);
config.swapBuffers(false);
surface->setConfiguration(config);
// For drawing
LPDIRECT3DDEVICE9 device = (LPDIRECT3DDEVICE9)surface->getDevice();
device->BeginScene();
// setup matrices and draw..
// the matrices and viewport will be preserved
surface->drawDU( rect.left, rect.bottom, rect.right, rect.top, true ) ;
device->EndScene();
device->Present( NULL, NULL, NULL, NULL );
