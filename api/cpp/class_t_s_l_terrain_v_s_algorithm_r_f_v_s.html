Detailed Description

A viewshed class, based on the RFVS[Franklin and Ray 1994] viewshed algorithm.

The following should be noted when using this algorithm: -The viewshed will be performed at the same resolution as the input data. If a lower resolution is requried, this must be exposed by the input object. -The viewshed will be performed on a circular area, defined by the maximum radius parameter. -The viewshed area will be limited to the input data extent, in order to provide a more efficient calculation. The extent will not be limited however if the center point of the viewshed is defined outside the input extent.

-Any location filters will only be checked for points along the viewshed's border. They will not be checked for every input sample. -Changing the maximum radius parameter will cause a slight difference in the viewshed output, due to the difference in spoke paths. This is noticeable in situations where an area is barely visible, and each terrain cell is different than its neighbours. -The viewshed operation will only set output data cells for the processed area. -Output data cells may be set more than once, as multiple spokes may cover the same location. When using this algorithm, particularly when combined with earth curvature corrections, a compositor such as TSLTerrainVSCompositor CumulativeVisibility should be used. This ensures that if a point is on the border of visibility, then it will be marked as visible.

Perform a line of sight calculation, between 2 points. If the end point is not visible from the start point then the 'blocked' point will be calculated. This is the point on the terrain which blocks the line of sight between the start and end points.

This method requires a valid input data object to be set on the viewshed parameters object.

Parameters
startXThe X coordinate of the start point, in map units.
startYThe Y coordinate of the start point, in map units.
startZThe height of the start point. This height value is absolute, and not relative to the ground.
endXThe X coordinate of the end point, in map units.
endYThe Y coordinate of the end point, in map units.
endZThe height of the end point. This height value is absolute, and not relative to the ground.
blockedXPopulated with the X coordinate of the 'blocked' point.
blockedYPopulated with the Y coordinate of the 'blocked' point.
blockedZPopulated with the height of the 'blocked' point.
ignoreNullIf true, the algorithm will ignore nulls in the input data. If false, the calculation will terminate if a null is encountered.
Returns
The following may be returned:
  • LOSResultVisible, if the end point is visible from the start point.
  • LOSResultBlocked, if the end point is not visible from the start point, and populates the 'blocked' point.
  • LOSResultNullData, if the calculation encountered a null height value. In this case the 'blocked' point will be set to the location of the null value.
