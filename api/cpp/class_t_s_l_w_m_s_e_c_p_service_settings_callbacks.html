Detailed Description

The TSLWMSECPServiceSettingsCallbacks should be inherited from in order to use the new OASIS 'Security Assertion Markup Language' (SAML) version 2.0 'Enhanced Client or Proxy' profile (ECP) functionality of the MapLink WMS Data Layer. This functionality allows the layer to communicate with WMS servers that require users to authorise themselves using the SAML security standard, so long as the 'Identity Providers' (IDP) and 'Service Providers' (SP) support the ECP profile.

It is recommended that users wishing to use the ECP functionality should familiarise themselves with both the SAML v2.0 standard and the ECP profile. Both of these are available from from OASIS (http://www.oasis-open.org).

As ECP uses the Secure Sockets Layer (SSL) protocol, certain options are available on the TSLFileLoaderRemote to relax rules used.

To access a WMS using ECP authentication the TSLWMSDataLayer's loadDataWithECP method should be used instead of loadData.

The order of that callbacks may be called is altered if using the ECP functionality of the data layer. As the layer must authorise itself with an IDP first, before proceeding with the normal WMS loading, the callbacks provided by this class will always be called before the callbacks of the base class, TSLWMSServiceSettingsCallback. The single exception to this is the onCapabilitiesLoadFailure callback which may be called anywhere throughout the load.

The new callback order is listed below:

-onChoiceOfIDP: If the federation metadata lists multiple IDPs that support the ECP profile, the user will be presented with a choice of the IDP to use.

-onCredentialsRequired: Unless the IDP has already been authenticated against in the current session, this callback will be made to request the username and password to provide to the IDP.

◆ onChoiceOfIDP()

virtual int TSLWMSECPServiceSettingsCallbacks::onChoiceOfIDP ( unsigned int numberOfIDPs,
const IDPOrganisation * organisations[],
const char * urls[],
const char * entityIDs[] )
pure virtual

This callback will be made when the SAML federation contains multiple Identity Providers (IDPs) that support the ECP profile and therefore can be used to authenticate against. The callback implementation should return a value to indicate which IDP should be used to authenticate against.

Parameters
numberOfIDPsThe number of IDPs that are available. This will be the size of the arrays passed in the remaining parameters.
organisationsAn array of Organisation objects that contain human readable information on the providing organisations of the IDPs.
urlsAn array of the URL end points for ECP communication with IDPs.
entityIDsAn array of the entityIDs of the IDPs. These can be used to uniquely identify the IDP.

Return the index of the IDP to use. Returning an invalid index will cause loading to fail.
